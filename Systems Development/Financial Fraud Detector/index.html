Financial Fraud Regression Model Web Application

If you've noticed the backdrop of this application, it represents Chicagoâ€”a city highlighted by Forbes as one of the major hubs for financial fraud in the United States. This choice is not accidental but a subtle nod to the subject matter of this web application, which I developed as a Master's student in Computer Information Systems at West Texas A&M University.

This web application, named the Fraud Detector, is designed to assess the potential for financial fraud within corporate data. It implements a logistic regression model based on the influential work of Kanapickiene and Grundiene (2015), which offers robust insights into financial fraud detection. If you possess company data and wish to evaluate the likelihood of fraud, you can input the data into the detector on the right side of the screen.

Below, you will find three useful buttons designed to help you familiarize yourself with the application's functionality:

  • Fraudulent Data Example: Automatically populates the form with data from a fraudulent company.
  • Non-Fraudulent Data Example: Loads data from a company without fraud.
  • Clear Form: Resets the form completely, allowing for a fresh start with new data.

Simply click on any of these options and then select "Verdict" to see how the application processes and analyzes the data. This practical tool is aimed at demonstrating both the application of advanced analytical techniques and the ease of user interaction with complex data systems.
